# PoE Arbitrage Dashboard 0.5.0

Браузерный дашборд для поиска потенциально выгодных операций в экономике Path of Exile 1.

Проект разворачивается на Cloudflare Pages. Интерфейс находится в `public`, серверные прокси — в `functions/api`.

## Что поддерживается

- 3 масла → 1 масло следующего тира;
- 3 эссенции → 1 эссенция следующего тира;
- комплект divination cards → фиксированное количество конкретной валюты;
- комплект divination cards → конкретная карта или конкретный фрагмент;
- комплект divination cards → конкретный scarab.

Экипировка и случайные награды намеренно не добавлены: их цена слишком сильно зависит от роллов, варианта предмета или случайного результата.

## Что добавлено в 0.5.0

- подкатегории внутри вкладки «Карточки»:
  - Все карточки;
  - Валюта;
  - Карты и фрагменты;
  - Скарабы;
- единый каталог фиксированных наград карточек;
- 8 проверенных цепочек «карточки → карта/фрагмент»;
- 4 проверенные цепочки «карточки → scarab»;
- дополнительные источники цен poe.ninja:
  - `Fragment`;
  - `UniqueMap`;
  - `Scarab`;
- необязательная загрузка новых категорий: если одна из них временно недоступна, масла, эссенции и валютные карточки продолжают работать;
- отдельная история цен для фрагментов, уникальных карт и scarab;
- подкатегория карточек в CSV-экспорте;
- 16 автоматических тестов.

Каталог карточек сверялся с набором данных `@navali/poe1-divination-cards` версии `3.28.2`, основанным на данных PoE Wiki.

## Какие карточки исключены

Не рассчитываются:

- случайная карта;
- случайный фрагмент;
- случайный scarab;
- generic-награды вроде `Scarab` или `Horned Scarab`;
- карты с особыми модификаторами, 100% Delirium или неоднозначной corrupted-версией;
- экипировка;
- случайные предметы.

Это сделано, чтобы не показывать заведомо ненадёжную прибыль.

## Как считается цена

При включённом параметре «Консервативный расчёт по двум источникам»:

```text
Цена покупки = max(poe.ninja, poe.watch)
Цена продажи = min(poe.ninja, poe.watch)
```

Затем применяются пользовательские коэффициенты:

```text
Затраты = цена покупки × размер комплекта × (1 + наценка покупки)
Продажа = цена награды × количество награды × (1 - скидка продажи)
Прибыль = продажа - затраты
ROI = прибыль / затраты × 100%
```

Если предмет отсутствует в poe.watch, расчёт продолжает использовать poe.ninja, а строка получает пометку «1 источник».

## Быстрый просмотр без API

После распаковки запустите в папке проекта:

```bash
python -m http.server 8000 --directory public
```

Откройте:

```text
http://localhost:8000/?demo=1
```

В демо-режиме есть примеры валютных карточек, карт/фрагментов и scarab.

## Полный локальный запуск

Требуется Node.js 20 или новее.

```bash
npm install
npm run dev
```

Wrangler обычно откроет сайт по адресу:

```text
http://localhost:8788
```

## Проверка API

```text
http://localhost:8788/api/leagues
http://localhost:8788/api/prices?league=Standard&type=Oil
http://localhost:8788/api/prices?league=Standard&type=Essence
http://localhost:8788/api/prices?league=Standard&type=DivinationCard
http://localhost:8788/api/prices?league=Standard&type=Currency
http://localhost:8788/api/prices?league=Standard&type=Fragment
http://localhost:8788/api/prices?league=Standard&type=UniqueMap
http://localhost:8788/api/prices?league=Standard&type=Scarab
http://localhost:8788/api/watch?league=Standard
```

Если один из необязательных endpoint `Fragment`, `UniqueMap` или `Scarab` временно не отвечает, главный дашборд должен продолжить работу и показать предупреждение в строке статуса.

## Развёртывание обновления

1. Распакуйте архив.
2. Откройте GitHub-репозиторий проекта.
3. Нажмите **Add file → Upload files**.
4. Перетащите всё содержимое папки проекта в корень репозитория.
5. Подтвердите замену файлов.
6. Нажмите **Commit changes**.
7. Дождитесь автоматического Deployment в Cloudflare Pages.
8. Обновите сайт через `Ctrl + F5`.

Важно загрузить обновлённые файлы:

```text
public/app.js
public/cards.js
public/core.js
public/index.html
public/styles.css
functions/api/prices.js
```

После проверки карт/фрагментов и scarab можно отдельно добавить:

## Ограничения

- poe.ninja и poe.watch дают агрегированные оценки, а не гарантированную цену покупки нужного количества;
- объём poe.watch является индикатором ликвидности, но не заменяет ручную проверку рынка;
- отдельные устаревшие или отсутствующие на рынке награды просто не создадут строку в таблице;
- формат сторонних API может измениться;
- дашборд ничего не покупает и не продаёт автоматически.

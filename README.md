# PoE Arbitrage Dashboard 0.6.0

Браузерный дашборд для поиска потенциально выгодных операций в экономике Path of Exile 1. Проект разворачивается на Cloudflare Pages: интерфейс находится в `public`, серверные прокси — в `functions/api`.

## Что поддерживается

- 3 масла → 1 масло следующего тира;
- 3 эссенции → 1 эссенция следующего тира;
- комплект divination cards → фиксированная валюта;
- комплект divination cards → конкретная карта или фрагмент;
- комплект divination cards → конкретный scarab;
- комплект divination cards → точный камень;
- комплект divination cards → конкретный уникальный предмет.

Случайные награды и обычная редкая экипировка намеренно не учитываются.

## Что добавлено в 0.6.0

### Диагностика фильтров

Над таблицей показывается:

- сколько операций рассчитано;
- сколько скрыто минимальной прибылью;
- сколько скрыто минимальным ROI;
- сколько скрыто стабильностью;
- сколько скрыто максимальным расхождением источников;
- сколько строк показано.

Кнопка **«Показать всё»** устанавливает:

```text
Мин. прибыль = 0
Мин. ROI = 0
Мин. стабильность = 1
Макс. расхождение = 0
```

При пустой таблице интерфейс сообщает, какой именно фильтр скрыл предложения.

### Карточки → камни

Добавлено 11 проверенных карточек с точными наградами. Сопоставление выполняется одновременно по:

- названию камня;
- уровню;
- качеству;
- состоянию corrupted.

Например, уровень 4 Empower Support не будет заменён ценой обычного камня первого уровня. Generic-награды вроде «случайный камень», «Transfigured Gem» или «Exceptional Support Gem» исключены.

### Карточки → уникальные предметы

Добавлена начальная выборка из 21 карточки, выдающей конкретный уникальный предмет. Используются только обычные неосквернённые версии без гарантированного шестилинка, специальных имплицитов или случайного уникального предмета.

У уникальных предметов характеристики могут иметь разные роллы, поэтому каталог получает пониженную оценку доверия и соответствующее пояснение.

### Подробная надёжность

Нажатие на оценку `XX/100` раскрывает причины:

- число исторических замеров;
- продолжительность прибыльного сигнала;
- один или два источника цены;
- расхождение poe.ninja и poe.watch;
- объём poe.watch;
- текущий ROI;
- точность типа награды;
- штраф за разброс роллов уникального предмета.

## Источники цен

- `poe.ninja` — основной источник кандидатов;
- `poe.watch` — второй источник, объём, изменение за 24 часа и история.

При включённом консервативном режиме:

```text
Цена покупки = max(poe.ninja, poe.watch)
Цена продажи = min(poe.ninja, poe.watch)
```

Если точный вариант отсутствует в poe.watch, строка продолжает рассчитываться по poe.ninja и получает пометку «1 источник».

## Быстрый просмотр без API

```bash
python -m http.server 8000 --directory public
```

Откройте:

```text
http://localhost:8000/?demo=1
```

Демо содержит примеры валюты, карт, фрагментов, scarab, точных камней и уникальных предметов.

## Полный локальный запуск

Требуется Node.js 20 или новее.

```bash
npm install
npm run dev
```

Обычно Wrangler открывает сайт по адресу:

```text
http://localhost:8788
```

## Развёртывание обновления

1. Распакуйте архив.
2. Откройте GitHub-репозиторий проекта.
3. Нажмите **Add file → Upload files**.
4. Перетащите всё содержимое папки проекта в корень репозитория.
5. Подтвердите замену файлов.
6. Нажмите **Commit changes**.
7. Дождитесь автоматического Deployment в Cloudflare Pages.
8. Обновите сайт через `Ctrl + F5`.

Особенно важно заменить:

```text
public/app.js
public/cards.js
public/core.js
public/index.html
public/styles.css
functions/api/prices.js
```

## Ограничения

- агрегированная цена не гарантирует покупку нужного количества по указанному курсу;
- цена уникального предмета зависит от роллов, даже когда название и версия совпадают;
- точный камень не отображается, если API не вернул его уровень, качество или corruption-состояние;
- отсутствующая на рынке награда просто не создаёт строку;
- формат сторонних API может измениться;
- дашборд ничего не покупает и не продаёт автоматически.

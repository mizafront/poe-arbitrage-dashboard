# PoE Arbitrage Dashboard 0.3.0

Браузерный дашборд для поиска потенциально выгодных операций в экономике Path of Exile 1:

- 3 масла → 1 масло следующего обычного тира;
- 3 эссенции → 1 эссенция следующего тира;
- полный комплект divination cards → фиксированное количество конкретной валюты.

Проект рассчитан на Cloudflare Pages: статический интерфейс находится в `public`, а функции-прокси — в `functions/api`.

## Что добавлено в 0.3.0

- вкладка «Карточки»;
- одновременная загрузка `DivinationCard` и `Currency` с poe.ninja;
- каталог из 43 карточек с детерминированной валютной наградой;
- учёт размера комплекта карточек;
- учёт количества валюты в награде;
- отдельная история цен карточки и награды;
- расчёт прибыли, ROI, бюджета, стабильности и доверия для карточных операций;
- демо-данные для проверки карточек без живого рынка;
- два дополнительных автоматических теста карточного расчёта.

Каталог первого этапа намеренно не включает случайную валюту, случайные scarabs, essences, уникальные предметы, карты, камни или награды с неоднозначным вариантом.

## Что уже работает

- получение актуального списка экономических лиг;
- постоянное добавление лиги `Standard`;
- получение цен Oil, Essence, DivinationCard и Currency;
- резервный переход на старый endpoint poe.ninja;
- консервативная цена покупки и быстрой продажи;
- прибыль, ROI и расчёт количества операций под заданный бюджет;
- локальная история цен в браузере за последние 7 дней;
- устойчивость возможности по последовательным замерам;
- изменение маржи относительно примерно часового снимка;
- приблизительный показатель доверия;
- автообновление каждые 5, 15 или 30 минут;
- экспорт отфильтрованных результатов в CSV;
- ручной ввод названия лиги.

## Быстрый просмотр интерфейса без API

После распаковки откройте терминал в папке проекта:

```bash
python -m http.server 8000 --directory public
```

Затем откройте:

```text
http://localhost:8000/?demo=1
```

Во вкладке «Карточки» должны появиться демонстрационные операции, в том числе The Fortunate и The Wrath.

## Полный локальный запуск

Требуется Node.js 20 или новее.

```bash
npm install
npm run dev
```

Wrangler покажет локальный адрес, обычно `http://localhost:8788`.

Проверка API:

```text
http://localhost:8788/api/leagues
http://localhost:8788/api/prices?league=Standard&type=Oil
http://localhost:8788/api/prices?league=Standard&type=Essence
http://localhost:8788/api/prices?league=Standard&type=DivinationCard
http://localhost:8788/api/prices?league=Standard&type=Currency
```

## Публикация обновления

1. Распакуйте архив.
2. Загрузите всё содержимое папки в корень существующего GitHub-репозитория с заменой файлов.
3. Нажмите `Commit changes`.
4. Cloudflare Pages автоматически создаст новое deployment.
5. После публикации выполните жёсткое обновление страницы: `Ctrl + F5`.

Настройки Cloudflare Pages остаются прежними:

- Framework preset: `None`;
- Build command: пусто;
- Build output directory: `public`.

## Как считается карточная возможность

```text
затраты = цена одной карточки × размер комплекта × (1 + наценка покупки)
продажа = цена одной единицы награды × количество награды × (1 − скидка продажи)
прибыль = продажа − затраты
ROI = прибыль / затраты × 100%
```

Пример:

```text
The Fortunate: 12 карточек
Награда: 2 Divine Orb

Цена карточки: 18c
Цена Divine Orb: 150c
Наценка покупки: 5%
Скидка продажи: 10%

Затраты = 18 × 12 × 1,05 = 226,8c
Продажа = 150 × 2 × 0,90 = 270c
Прибыль = 43,2c
```

## Откуда взят каталог карточек

Файл `public/cards.js` содержит только имена карточек, размеры комплектов и фиксированные валютные награды. Каталог подготовлен по данным PoE Wiki через пакет `@navali/poe1-divination-cards` версии 3.28.2. Изображения и полный текст карточек в проект не копируются.

При выходе 3.29 каталог нужно будет сверить с новой версией данных, если GGG изменит награды или добавит новые карточки.

## История и стабильность

История хранится в `localStorage` только в текущем браузере. Снимок карточной операции содержит отдельно:

- цену одной карточки;
- цену одной единицы валютной награды.

Например, `3 / 5` означает, что операция оставалась прибыльной в трёх последних последовательных замерах при пяти доступных замерах.

## Ограничения первого этапа

- poe.ninja показывает агрегированную оценку, а не гарантированную цену исполнения;
- программа не проверяет, можно ли купить весь комплект карточек одним объёмом;
- программа не проверяет реальную цену быстрой продажи награды в Currency Exchange;
- карточки со случайными и предметными наградами пока исключены;
- если poe.ninja не отслеживает конкретную валюту, соответствующая карточка не появится;
- API poe.ninja может меняться;
- запросы к источнику кэшируются на пять минут.

## Что проверять после публикации

1. Открывается `/api/prices?league=Standard&type=DivinationCard`.
2. Открывается `/api/prices?league=Standard&type=Currency`.
3. Во вкладке «Карточки» появляются строки.
4. У карточки правильный размер комплекта.
5. У награды правильное количество валюты.
6. При ручном расчёте затраты и продажа совпадают с таблицей.

## Следующий этап карточек

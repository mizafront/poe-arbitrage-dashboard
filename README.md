# PoE Arbitrage Dashboard 0.4.0

Браузерный дашборд для поиска потенциально выгодных операций в экономике Path of Exile 1.

Поддерживаются:

- 3 масла → 1 масло следующего тира;
- 3 эссенции → 1 эссенция следующего тира;
- комплект divination cards → фиксированное количество конкретной валюты.

Проект разворачивается на Cloudflare Pages. Интерфейс находится в `public`, серверные прокси — в `functions/api`.

## Что добавлено в 0.4.0

Реализованы сразу три этапа двойной проверки:

1. Подключён poe.watch как второй источник цен.
2. Добавлены расхождение источников и новая оценка надёжности.
3. Добавлены объём poe.watch, изменение за 24 часа и мини-график за 7 дней.

Также добавлены:

- консервативный расчёт по двум источникам;
- фильтр максимального расхождения цен;
- сортировка по расхождению и объёму;
- счётчик операций, подтверждённых двумя источниками;
- выгрузка цен обоих источников в CSV;
- автоматический переход на poe.ninja, если poe.watch недоступен;
- новая функция `/api/watch`;
- 13 автоматических тестов.

## Как считается цена

При включённом параметре «Консервативный расчёт по двум источникам»:

```text
Цена покупки = max(poe.ninja, poe.watch)
Цена продажи = min(poe.ninja, poe.watch)
```

После этого применяются пользовательские коэффициенты:

```text
Затраты = цена покупки × количество × (1 + наценка покупки)
Продажа = цена результата × количество награды × (1 - скидка продажи)
Прибыль = продажа - затраты
ROI = прибыль / затраты × 100%
```

Если предмет отсутствует в poe.watch, расчёт продолжает использовать poe.ninja, а строка получает пометку «1 источник».

## Расхождение источников

Расхождение считается симметрично:

```text
|Ninja - Watch| / ((Ninja + Watch) / 2) × 100%
```

Практический ориентир:

- до 7% — хорошее совпадение;
- 7–15% — допустимое расхождение;
- более 15% — нужна ручная проверка;
- отсутствие poe.watch — сигнал не подтверждён вторым источником.

## Быстрый просмотр без API

После распаковки запустите в папке проекта:

```bash
python -m http.server 8000 --directory public
```

Откройте:

```text
http://localhost:8000/?demo=1
```

Демо содержит цены двух источников, объёмы, изменение за 24 часа и историю за 7 дней.

## Полный локальный запуск

Требуется Node.js 20 или новее.

```bash
npm install
npm run dev
```

Wrangler обычно откроет сайт по адресу:

```text
http://localhost:8788
```

## Проверка API

```text
http://localhost:8788/api/leagues
http://localhost:8788/api/prices?league=Standard&type=Oil
http://localhost:8788/api/prices?league=Standard&type=Essence
http://localhost:8788/api/prices?league=Standard&type=DivinationCard
http://localhost:8788/api/prices?league=Standard&type=Currency
http://localhost:8788/api/watch?league=Standard
```

На опубликованном сайте:

```text
https://poe-arbitrage-dashboard.pages.dev/api/watch?league=Standard
```

Если `/api/watch` возвращает `502`, poe.watch не ответил ни через один из предусмотренных endpoint. Главный дашборд при этом должен продолжить работу только по poe.ninja и показать предупреждение.

## Развёртывание обновления

1. Распакуйте архив.
2. Откройте репозиторий GitHub проекта.
3. Нажмите **Add file → Upload files**.
4. Перетащите всё содержимое папки проекта в корень репозитория.
5. Подтвердите замену файлов.
6. Нажмите **Commit changes**.
7. Дождитесь автоматического Deployment в Cloudflare Pages.
8. Обновите сайт через `Ctrl + F5`.

Новый файл, который обязательно должен попасть в репозиторий:

```text
functions/api/watch.js
```

## Ограничения

- poe.ninja и poe.watch дают агрегированные оценки, а не гарантированную цену покупки нужного количества.
- Объём poe.watch является индикатором ликвидности, но не заменяет проверку рынка в игре.
- Формат сторонних API может измениться. Парсер poe.watch сделан гибким и принимает несколько форматов, однако после крупных обновлений источника может потребоваться адаптация.
- Дашборд ничего не покупает и не продаёт автоматически.

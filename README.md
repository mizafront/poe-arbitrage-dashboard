# PoE Arbitrage Dashboard 0.2

Браузерный дашборд для поиска потенциально выгодных рецептов Path of Exile 1:

- 3 масла → 1 масло следующего обычного тира;
- 3 эссенции → 1 эссенция следующего тира.

Проект рассчитан на Cloudflare Pages: статический интерфейс находится в `public`, а функции-прокси — в `functions/api`.

## Что уже работает

- получение актуального списка экономических лиг;
- получение цен Oil и Essence с текущего endpoint poe.ninja;
- резервный переход на старый endpoint poe.ninja, если новый временно недоступен;
- консервативная цена покупки и быстрой продажи;
- прибыль, ROI и расчёт количества операций под заданный бюджет;
- локальная история цен в браузере за последние 7 дней;
- устойчивость возможности по последовательным замерам;
- изменение маржи относительно примерно часового снимка;
- приблизительный показатель доверия;
- автообновление каждые 5, 15 или 30 минут;
- экспорт отфильтрованных результатов в CSV;
- ручной ввод названия лиги, если список ещё не успел обновиться.

## Быстрый просмотр интерфейса без API

После распаковки откройте терминал в папке проекта:

```bash
python -m http.server 8000 --directory public
```

Затем откройте:

```text
http://localhost:8000/?demo=1
```

Демо-режим использует искусственные цены и нужен только для проверки интерфейса.

## Полный локальный запуск

Требуется Node.js 20 или новее.

```bash
npm install
npm run dev
```

Wrangler покажет локальный адрес, обычно `http://localhost:8788`.

Проверка API:

```text
http://localhost:8788/api/leagues
http://localhost:8788/api/prices?league=Mirage&type=Oil
http://localhost:8788/api/prices?league=Mirage&type=Essence
```

## Публикация через Cloudflare Pages и GitHub

1. Создайте пустой репозиторий GitHub.
2. Загрузите **содержимое этой папки** в корень репозитория.
3. В Cloudflare откройте **Workers & Pages → Create → Pages → Connect to Git**.
4. Выберите репозиторий.
5. Настройки сборки:
   - Framework preset: `None`;
   - Build command: оставить пустым;
   - Build output directory: `public`.
6. Запустите Deploy.

После публикации проверьте:

```text
https://ВАШ-САЙТ.pages.dev/api/leagues
https://ВАШ-САЙТ.pages.dev/api/prices?league=Mirage&type=Oil
```

## Как считается возможность

```text
затраты = цена сырья × 3 × (1 + наценка покупки)
продажа = цена результата × (1 − скидка продажи)
прибыль = продажа − затраты
ROI = прибыль / затраты × 100%
```

«Прибыль на бюджет» не учитывает реальный доступный объём. Она показывает, сколько операций теоретически помещается в указанный бюджет.

## История и стабильность

История хранится в `localStorage` только в том браузере, где открыт сайт. Дашборд сохраняет до 7 дней и не отправляет историю на сервер.

- `3 / 5` означает: рецепт был прибыльным в трёх последних последовательных замерах, всего доступно пять замеров.
- При фильтре «3 замера» после первого запуска таблица может быть пустой. Для первоначальной проверки поставьте «1 замер».

## Ограничения

- poe.ninja показывает агрегированную оценку, а не гарантирует покупку нужного количества по указанной цене;
- текущий API poe.ninja не является стабильным публичным контрактом и может измениться;
- запросы к источнику кэшируются на пять минут;
- история в браузере теряется после очистки данных сайта или при переходе в другой браузер;
- специальные масла не включены в обычную цепочку 3 → 1;
- особые эссенции Horror, Delirium, Hysteria и Insanity не включены в обычную цепочку повышения.
